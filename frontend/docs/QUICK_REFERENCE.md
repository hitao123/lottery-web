# 婚礼抽奖现场速查

## 现场开始前

1. 在项目根目录复制 `.env.example` 为 `.env`，设置 `LOTTERY_ADMIN_TOKEN`。
2. 启动 Docker Compose 后，打开大屏页面并进入“控制面板”。
3. 输入同一操作密码并点击“连接”，确认已恢复正确的名单与中奖记录。
4. 导入名单前会显示二次确认；支持 `1-200` 范围或换行/逗号分隔的编号列表。

## 快捷键

| 按键 | 条件 | 操作 |
| --- | --- | --- |
| Space | 空闲且仍有候选人 | 开始洗牌 |
| S | 洗牌中 | 请求后端安全抽取并锁定唯一结果 |
| Enter / N | 公布结果后 | 进入下一轮 |
| R | 空闲或公布结果后 | 二次确认后重置全部中奖记录 |
| F | 任意时间 | 切换全屏 |

## 抽奖规则

- 后端使用 `crypto/rand` 产生中奖结果，并将结果、轮次和名单原子保存到数据卷。
- 页面只展示后端返回的结果；网络失败或超时会显示错误，**不会**改用浏览器随机数。
- 请求进行中会进入“正在安全抽取”状态，重复按 S 不会触发第二次抽奖。
- 撤销、重置、重新导入都会同步到后端。刷新页面或重启容器后会恢复保存的状态。

## 现场故障处理

- 出现“需要操作密码”：在控制面板输入 `.env` 中的 `LOTTERY_ADMIN_TOKEN` 后点击“连接”。
- 出现请求失败：不要重复按 S；检查 API 容器状态后点击连接刷新名单，再继续抽奖。
- 来宾不在现场：下一轮前，在中奖名单点击 × 撤销，再重新抽取。
- 需要交付记录：在中奖名单点击“导出 CSV”。
